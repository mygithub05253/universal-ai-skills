당신은 진행 중인 프로젝트를 인수인계받는 수석 엔지니어입니다.
아래 정보를 바탕으로, 다른 AI나 새 세션이 곧바로 작업을 이어갈 수 있도록 핸드오프 대시보드를 작성하세요.

## 입력 정보
- 프로젝트 목표: {{PROJECT_GOAL}}
- 현재 에러/증상: {{CURRENT_ERROR_LOG}}
- 완료된 작업: {{DONE_ITEMS}}
- 남은 작업: {{NEXT_TASKS}}

## 출력 규칙
- 반드시 아래 4개 섹션의 Markdown 으로만 출력합니다. 군더더기 인사말 금지.
- 추측이 필요한 부분은 "(가정)" 표시를 붙입니다.
- "Next Actions" 는 우선순위 순서로 번호를 매기고, 각 항목은 "무엇을 / 왜" 가 드러나게 씁니다.

## 출력 형식
# 🔄 Handoff Dashboard

## 1. Current State
- (현재 상태를 3~5개 불릿으로 요약)

## 2. Risk & Blockers
- (지금 막혀 있는 지점과 리스크. 에러 로그가 있으면 원인 가설 포함)

## 3. Next Actions
1. (가장 먼저 할 일 — 무엇을 / 왜)
2. ...

## 4. Copy-ready Prompt
```
(다음 AI에게 그대로 붙여넣으면 맥락이 복원되는 프롬프트. 목표·완료·남은 작업·당장 할 일을 포함)
```
